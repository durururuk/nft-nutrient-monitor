# 배양액 도징 알고리즘 설계 문서

> **요약**: 이 문서는 「설계서 v2」의 A·B 양액 응답곡선 실험을 기반으로,
> 탱크의 현재 EC·pH를 읽어 **A·B 투입량(mL)을 역산하는 도징 알고리즘**의 수학 모델과 구현 방향을 정리한 것입니다.
>
> **핵심 메시지**: 알고리즘의 *식(구조)*은 실험 데이터 없이도 지금 확정할 수 있습니다.
> 실험이 채우는 것은 식이 아니라 **식에 들어갈 계수(숫자)**뿐입니다.
> 따라서 식을 먼저 구축해두고, 실험이 끝나면 계수만 교체하는 방식으로 진행합니다.

---

## 1. 개요 — "식은 지금, 숫자는 실험 후"

도징 알고리즘은 두 부분으로 나뉩니다.

| 구성 | 내용 | 지금 가능한가? |
| --- | --- | --- |
| **식의 구조** | EC·pH로부터 A·B 투입량을 계산하는 수학적 형태 | **예 — 지금 확정** |
| **계수(숫자)** | 식 안에 들어갈 응답 계수 | 아니오 — 실험으로 측정 |

이 둘을 분리해 설계하면, 실험 전에 알고리즘·소프트웨어를 모두 완성해두고
실험이 끝난 뒤에는 **계수 숫자만 갈아끼우는 것**으로 시스템이 완성됩니다.
코드 로직은 다시 손대지 않습니다.

---

## 2. 실험 → 계수 매핑

「설계서 v2」의 세 실험은 각각 아래 계수를 산출하기 위한 것입니다.
모든 계수는 **물 1L당 양액 1mL를 넣었을 때의 변화량(mL/L 단위)**으로 저장합니다.
이렇게 하면 실험 용량(4L)과 실제 운용 탱크 용량이 달라도 같은 계수를 그대로 씁니다.

| 계수 | 의미 | 도출 실험 |
| --- | --- | --- |
| `a_ec_A` | A액 1mL/L당 **EC 상승량** | 실험 1 (A 단독) 기울기 |
| `a_ph_A` | A액 1mL/L당 **pH 변화량** (석회 성분 → 상승) | 실험 1 |
| `a_ec_B` | B액 1mL/L당 **EC 상승량** | 실험 2 (B 단독) 기울기 |
| `a_ph_B` | B액 1mL/L당 **pH 변화량** (인산 성분 → 하강) | 실험 2 |
| `c_ec` | A·B를 함께 넣을 때 EC의 **비선형 보정** | 실험 3 (A+B 혼합) |
| `c_ph` | A·B를 함께 넣을 때 pH의 **비선형 보정** | 실험 3 |

> 즉 실험 1·2는 각 양액의 단독 효과를, 실험 3은 **둘이 섞였을 때의 상호작용**을 측정합니다.

---

## 3. 도징 수학 모델

### 기호 정의

| 기호 | 의미 |
| --- | --- |
| `V` | 운용 탱크 용량 (L) |
| `A`, `B` | A액·B액 투입량 (mL) — **구하려는 값** |
| `x_A = A/V`, `x_B = B/V` | 단위 부피당 투입량 (mL/L) |
| `ΔEC = 목표EC − 현재EC` | 끌어올려야 할 EC |
| `ΔpH = 목표pH − 현재pH` | 보정해야 할 pH |

### 모델 식 (상호작용항 포함)

$$
\Delta EC = a_{ec,A}\,x_A + a_{ec,B}\,x_B + c_{ec}\,(x_A x_B)
$$

$$
\Delta pH = a_{ph,A}\,x_A + a_{ph,B}\,x_B + c_{ph}\,(x_A x_B)
$$

- 앞의 두 항은 **각 양액의 단독 효과**(실험 1·2),
- 마지막 `(x_A · x_B)` 항은 **둘이 함께 있을 때의 비선형 보정**(실험 3)입니다.

이는 EC와 pH **두 개의 식**, A와 B **두 개의 미지수**로 이루어진 연립방정식이며,
풀면 목표 EC·pH를 동시에 달성하는 A·B 투입량이 한 번에 나옵니다.
(EC만 맞추는 게 아니라 pH까지 능동적으로 제어합니다.)

### 풀이 방법

상호작용항 때문에 단순 대입으로는 풀리지 않으므로 2단계로 계산합니다.

1. **선형 초기해**: 상호작용항을 0으로 두면 닫힌 형태(2×2 행렬)로 즉시 풀립니다.
2. **반복 보정**: 그 초기해에서 출발해 상호작용항을 포함한 정확한 해로 2~3회 수렴시킵니다.
   상호작용은 작은 보정값이라 매우 빠르게 수렴하며, 외부 수치 라이브러리 없이 계산합니다.

### 투입 순서 가정

이 모델은 실제 운용 방식인
**「물 + A액 투입 → 충분히 교반 → B액 추가 투입」의 순차 도징**을 가정합니다.
이는 「설계서 v2」 실험 3의 측정 절차(A 먼저 1mL → 교반 → B 1mL → 교반 → 정치 → 측정)와
**동일한 조건**이므로, 실험에서 얻은 계수가 실제 운용에 그대로 적용됩니다.

---

## 4. 왜 혼합 실험(실험 3)이 반드시 필요한가

만약 단순히 "A 단독 효과 + B 단독 효과 = 혼합 효과"라고 가정한다면 실험 3은 불필요합니다.
그러나 실제로는 그렇지 않습니다.

- A액의 **석회(Ca)** 성분과 B액의 **인산(P)** 성분이 만나면 이온이 상호작용하고,
- pH는 **완충작용** 때문에 단순 합산으로 예측되지 않으며,
- 경우에 따라 침전이 생겨 EC가 예상보다 낮아질 수 있습니다.

이 **단순 합산과 실제 혼합의 차이**가 바로 `c_ec`, `c_ph` 계수입니다.
실험 3을 빼면 모델은 단순 선형합만 알게 되어, A와 B를 함께 투입하는
실제 운용 구간에서 오차가 누적됩니다.
따라서 실험 3은 정확도를 위한 **필수 입력**이며, 본 모델에 정식으로 반영되어 있습니다.

> 보조 효과: 실험 4(재현성 검증)에서 얻는 측정 표준편차는 도징 알고리즘의
> **데드밴드(허용 오차)** 폭을 정하는 근거로 활용합니다.

---

## 5. 부피 정규화와 탱크 용량

- 모든 계수는 **mL/L 단위**로 저장하므로, 실험(4L)과 운용 탱크의 용량이 달라도 그대로 사용합니다.
- 실제 투입량은 마지막에 `A = x_A · V`, `B = x_B · V`로 탱크 용량을 곱해 환산합니다.
- **운용 탱크 용량은 아직 미정**이므로, 소프트웨어에서 하나의 설정값(변수)으로 두고
  확정되면 숫자만 입력합니다. 알고리즘은 영향받지 않습니다.

---

## 6. 물리적 제약과 안전장치

식만으로는 비현실적인 답이 나올 수 있어, 다음 제약을 모델에 내장합니다.

| 제약 | 내용 |
| --- | --- |
| **음수 투입 불가** | 양액은 빼낼 수 없으므로 계산상 음수가 나오면 0으로 처리하고, EC를 우선해 나머지 한쪽만 재계산 |
| **데드밴드** | 목표와의 차이가 측정 오차(±0.1) 수준이면 도징하지 않음 (불필요한 미세 투입 방지) |
| **1회 투입 상한** | 한 번에 넣는 최대량을 제한, 초과 시 나눠서 투입하도록 안내 |
| **측정 범위 외 경고** | 실험으로 검증된 범위(표준 0~1.6배)를 벗어나면 "범위 밖 — 재측정 권장" 경고 표시 |

또한 도징 후에는 **교반 → 안정화 → 재측정**을 거쳐 다시 판단하는 폐루프로 운용합니다.

---

## 7. 계수 미정 상태에서의 동작과 향후 교체

- 실험 전에는 계수를 **설계서 예시값/임시값(placeholder)**으로 채워 둡니다.
  식은 정상 작동하며, 다만 결과에 **"실험 계수 미반영"** 표시가 함께 나옵니다.
- 특히 상호작용 계수 `c_ec`, `c_ph`는 실험 3 전까지 **0**으로 두므로,
  이 동안 모델은 자연스럽게 단순 선형 모델로 동작합니다.
- 실험 완료 후에는 측정값을 넣으면 6개 계수를 자동 산출하는 보조 함수를 통해
  **설정 파일의 숫자 6개와 탱크 용량만 교체**합니다.
- **코드 로직은 전혀 바뀌지 않습니다.**

## 8. 구현 로드맵 (교수님 검토 후 진행)

검토가 끝나면 아래 순서로 코드를 작성합니다.

1. **계수·설정 정의** — 6개 응답 계수, 탱크 용량, 데드밴드·상한 등 모든 숫자를 한곳에 모음.
2. **도징 엔진** — 위 연립방정식 풀이 + 제약 처리. 하드웨어와 무관한 순수 계산 모듈.
3. **계수 산출 보조 함수** — 실험표를 넣으면 계수를 계산해 주는 도구(실험 후 사용).
4. **API 연동** — 현재 센서값·목표값을 받아 도징량을 계산해 응답하는 엔드포인트.
   (실제 펌프 구동은 다음 단계, 지금은 계산 결과만 반환)
5. **화면 연동** — 도징 버튼을 누르면 "A ○mL + B ○mL 권장"이 표시되도록 연결.
6. **검증** — 알려진 계수로 만든 가상 시나리오를 역으로 풀어 원래 값이 복원되는지 확인.

## 참고 문헌 / 출처

> 본 도징 모델의 식은 특정 단일 논문을 인용한 것이 아니라, **이미 검증된 표준 기법 세 가지를 조합**한 것입니다.
> 각 구성요소의 근거를 아래에 정리합니다. (모든 링크는 작성 시점에 실재·관련성을 확인했습니다.)

**1. EC ↔ 양액 농도의 선형 응답** — 식의 `a_ec_A·x_A`, `a_ec_B·x_B` 항의 근거

- [Building a model to predict EC in hydroponic nutrient solutions — Science in Hydroponics](https://scienceinhydroponics.com/2020/07/building-a-model-to-predict-ec-in-hydroponic-nutrient-solutions.html)
  수경재배 양액에서 EC를 각 염(salt)의 **선형 기여 합**으로 모델링하는 접근. 본 설계가 A·B 양액별 EC 기여 계수를 따로 두는 것과 동일한 원리이며, "염마다 EC 기여도가 다르다"는 점도 함께 다룹니다(→ 실험 1·2로 계수를 직접 측정하는 이유).

**2. 혼합 상호작용항 (이중선형) / 반응표면법(RSM)** — 식의 `c·(x_A·x_B)` 항의 근거

- [Response Surface Methodology — JMP Statistics Knowledge Portal](https://www.jmp.com/en/statistics-knowledge-portal/design-of-experiments/response-surface-methodology)
  두 인자를 함께 변화시킬 때 단순 합으로 설명되지 않는 **2요인 상호작용항(two-factor interaction term)** 을 회귀 모델에 포함하는 표준 방법. 본 설계의 `c·(x_A·x_B)` 항(실험 3에서 도출)이 정확히 이 cross-term에 해당합니다.

**3. 비선형 연립방정식의 Newton-Raphson 풀이** — 상호작용항 포함 시 A·B 역산 방법의 근거

- [Newton-Raphson Method (Multivariate) — Harvey Mudd College](https://pages.hmc.edu/ruye/MachineLearning/lectures/ch2/node7.html)
  여러 미지수의 비선형 연립방정식을 **야코비안(Jacobian) 행렬**로 반복 수렴시켜 푸는 표준 수치해법. 상호작용항 때문에 닫힌형이 아닌 본 모델의 EC·pH 연립식을 A·B에 대해 푸는 데 사용합니다(`dosing.py`의 `_newton_solve`).

> 요약: **선형 응답 모델 + 2요인 상호작용항 + Newton-Raphson 수치해법**의 결합.
> 세 기법 모두 각 분야에서 표준으로 확립된 방법이며, 본 시스템은 이를 NFT 양액 도징 문제에 적용한 것입니다.

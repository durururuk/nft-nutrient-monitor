## 0. 요약 (Executive Summary)

- 물 4L 기준 A액/B액 단독·혼합·재현성 응답곡선을 실측하고, 도징 알고리즘의 **6개 응답 계수**를 도출했다.
- EC는 투입량에 대해 매우 선형(R² > 0.99, 표준 구간)이며, 계수 신뢰도가 높다.
- **가설 반증**: A액이 석회 성분으로 pH를 올릴 것이라는 예상과 달리, 실측상 **A액은 pH에 거의 영향이 없고(pH 중립), pH를 움직이는 것은 B액뿐**이다. → 양액 투입으로 pH를 **내릴 수만 있고 올릴 수는 없다.**
- EC·pH 역산 알고리즘은 **수학적으로 건전**하며 도달 가능 영역 내에서 목표를 정확히 재현한다.
- 그러나 **EC·pH만으로는 영양 성분 밸런스를 제어할 수 없다.** 따라서 도징은 **A:B = 1:1 정배합으로 EC만 맞추고, pH는 별도 산/염기로, 성분 드리프트는 운용 규칙(순환식이면 1~3개월 주기 교체 + 보충, 또는 배액식)으로 관리**하는 것이 유일하게 정직한 운용 방식이다.
- **이온 선택성(NPK) 센서는 이 규모에서 비현실적**이다: 싼 센서는 사실상 EC 추정치(가짜)이고, 진짜 ISE는 이온당 100만 원대이며, 정작 핵심 성분인 **인산(P)은 실시간 측정 수단 자체가 없다.**

## 1. 실험 개요

| 항목 | 조건 |
| --- | --- |
| 사용 배양액 | 물푸레 1호 A액 / B액 |
| 실험 베이스 | 수돗물 4L |
| 표준 1배 도징량 | A액 5mL + B액 5mL (20L 기준 25mL의 1/5) |
| 투입 스텝 | 1.0mL 단위, 0~8mL (표준 0~1.6배) |
| 측정 절차 | 투입 → 30초 교반 → 2분 정치 → EC·pH 측정 |
| 센서 정밀도 | EC ±100 μS/cm, pH ±0.1 (저가형 아두이노 연결형) |

실험 1(A단독), 2(B단독), 3(A+B 혼합), 4(재현성 3회)를 **하루에 연속 수행**하여 수원(수돗물) 조건 편차를 최소화했다.

## 2. 실험 결과

### 2.1 캘리브레이션·베이스라인

- pH 2점 보정: 6.86 → 6.91, 4.0 → 4.03 (오차 ±0.05 이내, 정상)
- EC 보정: 1410 표준액 → 1413 (정상)
- 베이스라인(수돗물): 실험별 EC 486 / 460 / 446 μS/cm, pH 7.85 / 7.96 / 7.81
  → 실험마다 새 수돗물이라 소폭 차이. 상승량(Δ) 계산은 항상 해당 실험 자신의 베이스라인 기준.

### 2.2 응답곡선 (표준 1배 = 5mL 지점 요약)

| 실험 | 베이스 EC → 1배 EC | ΔEC | 베이스 pH → 1배 pH | ΔpH |
| --- | --- | --- | --- | --- |
| A 단독 | 486 → 1252 | **+766** | 7.85 → 7.85 | **±0** |
| B 단독 | 460 → 844 | **+384** | 7.96 → 6.71 | **-1.25** |
| A+B 혼합 | 446 → 1634 | **+1188** | 7.81 → 6.58 | **-1.23** |

- **EC 선형성**: 표준 구간(0~1.0배) R² — A 0.999 / B 0.998 / 혼합 0.9995. 매우 양호.
- **1.2배 이상 포화 경향**: 혼합 EC 증가폭이 6~8스텝에서 둔화(255→…→142→67→58). 1회 투입 상한은 보수적으로 잡을 것.

### 2.3 재현성 (실험 4)

| 스텝 | ΔEC 표준편차 | pH 표준편차 |
| --- | --- | --- |
| 1.0배 | 16.2 μS/cm | 0.04 |
| 1.6배 | 6.7 μS/cm | 0.04 |

→ 기준(EC ±150, pH ±0.15) 대비 여유 있게 **합격**. 계수 신뢰 가능, 재보정 불필요.

### 2.4 도출된 도징 계수

물 1L당 양액 1mL를 넣을 때의 변화량(**mL/L 정규화 단위**, mS/cm 기준). 실험 용량(4L)과 운용 탱크 용량이 달라도 그대로 사용:

| 계수 | 값 | 의미 | 출처 |
| --- | --- | --- | --- |
| `a_ec_A` | **0.6128** | A액 1mL/L당 EC 상승 | 실험 1 |
| `a_ph_A` | **≈ 0.0** | A액 1mL/L당 pH 변화 | 실험 1 |
| `a_ec_B` | **0.3072** | B액 1mL/L당 EC 상승 | 실험 2 |
| `a_ph_B` | **≈ -1.0** | B액 1mL/L당 pH 변화 | 실험 2 |
| `c_ec` | **+0.0243** | 혼합 EC 비선형 보정 | 실험 3 |
| `c_ph` | **+0.0128** | 혼합 pH 비선형 보정 | 실험 3 |

- **EC 기여율**: A 64.5% / B 32.3% / 상호작용 +3.3%. (A단독+B단독=1150 μS/cm 대비 실측 혼합 1188 μS/cm → 미세한 양의 시너지)
- **표준 1배 도달점**: EC 1634 μS/cm(1.63 mS/cm), pH 6.58 — 상추 정상 범위 내이며 운용 목표점으로 적합.

## 3. 도징 알고리즘

### 3.1 모델 (EC·pH 연립 역산)

현재 EC·pH를 읽어 목표까지 필요한 A·B 투입량을 역산한다. `xA = A/V`, `xB = B/V` (mL/L):

```
ΔEC = a_ec_A·xA + a_ec_B·xB + c_ec·(xA·xB)
ΔpH = a_ph_A·xA + a_ph_B·xB + c_ph·(xA·xB)
```

- 2개 식(EC·pH) × 2개 미지수(A·B)의 연립방정식.
- 상호작용항 때문에 닫힌형이 아니므로: 선형 초기해 → Newton 반복 보정(2~3회 수렴).
- 구현: [dosing.py](../dosing.py)

### 3.2 알고리즘 검증 결과 (수학적 건전성)

- **유일해 존재**: 연립 행렬식 det = -0.6128 ≠ 0.
- **역검증(round-trip)**: 산출된 도징량을 순방향 모델에 다시 넣으면 목표를 정확히 재현
  (50L 탱크 기준 EC 2.000 / pH 6.500 일치).
- 결론: **도달 가능한 범위 안에서는 알고리즘이 정확하게 동작한다.**

### 3.3 핵심 발견 — pH 비대칭성

| 액 | EC | pH |
| --- | --- | --- |
| A액 | 올림 | **중립 (≈0)** |
| B액 | 올림 | 내림 |

- A액은 "순수 EC 조종간", B액은 "EC·pH가 묶인 조종간"이다.
- 투입량은 뺄 수 없으므로(xA, xB ≥ 0), **물리적 도달 영역은 `EC 상승 & pH 하강` 사분면뿐**이다.
- 즉 **양액으로 pH를 올리는 것은 불가능**하며, 필요 시 pH-up(염기)을 별도로 써야 한다.
- 이에 따라 알고리즘에 경고를 보강함: ① pH 상향 필요 시 "양액으로 상향 불가" 안내, ② EC는 목표인데 pH만 이탈한 경우 침묵하지 않고 사유 표시.

> ⚠️ **모델 한계**: `a_ph_B`(-1.0)는 강한 비선형 완충 곡선의 **평균 기울기**다. 베이스라인에서 먼 고농도 상태에서는 pH 예측 정확도가 떨어지므로, **도징 → 재측정 → 재계산의 폐루프**로 보완한다.

## 4. 왜 A:B = 1:1 정배합이어야 하는가 (영양 밸런스)

### 4.1 EC·pH는 "성분 밸런스"를 보지 못한다

- **EC**는 총 이온량(합계)만, **pH**는 산도만 측정한다. **어떤 성분이 얼마나** 있는지는 알 수 없다.
- 따라서 EC·pH가 동일해도 성분 비율이 전혀 다른 용액이 무수히 많다.

현재 2×2 알고리즘은 EC·pH 두 숫자만 맞추면 **A:B 비율이 아무 값이나 나오게** 둔다. 실측 시뮬레이션 예:

| 상황 | 산출 A:B | 배합비 |
| --- | --- | --- |
| 정상(EC낮·pH높) | 32.8 : 15.1 | 약 2.2 : 1 |
| pH 이미 목표 | 40.8 : 0 | A만 |

→ 정답 배합(1:1)에서 크게 벗어난다. **EC·pH는 맞지만 속(질소:인산:칼리:석회:고토 비율)이 깨진 용액**이 나온다.

### 4.2 정배합에서는 pH가 EC에 종속된다

1:1로 희석하면 실측 곡선상 (EC 446, pH 7.81) → (1634, 6.58) → (1901, 6.22)로,
**EC가 정해지면 pH도 자동으로 결정된다.** 밸런스를 지키면서 pH를 독립적으로 고를 자유도가 없다.
→ 알고리즘이 pH를 따로 맞추려고 B를 더 넣거나 A만 넣는 순간 = **정배합 이탈 = 밸런스 파괴**.

### 4.3 선택적 흡수와 드리프트 (관측 불가 문제)

상추는 성분을 균일하게 흡수하지 않는다(선택적 이온 흡수). 성분표 기준:

| 성분 | 출처 | 경향 |
| --- | --- | --- |
| 칼슘(석회) | **A액 전용** | 느리게 흡수 → **누적** |
| 인산·고토 | **B액 전용** | 빠르게 흡수 → **고갈** |
| 질소·칼리 | A·B 공통 | 빠르게 흡수 |

→ 시간이 지나면 탱크 성분비가 틀어진다(드리프트). **1:1 보충으로도 이 드리프트 자체는 막지 못한다.**

그러나 결정적으로 — **이 드리프트는 EC·pH로 관측할 수 없다.** 관측 못 하는 것은 어떤 알고리즘도 능동 교정할 수 없다. 현재 2×2가 비율을 흔드는 것은 드리프트 교정이 아니라 **pH라는 엉뚱한 신호를 보고 넣는 비율까지 망가뜨리는** 최악의 조합이다.

### 4.4 결론 — 정직한 운용 방식

| 역할 | 방법 |
| --- | --- |
| **EC 유지** | A:B = 1:1 정배합으로 목표 EC까지 투입 (넣는 비율 보존) |
| **pH 조절** | A·B가 아니라 별도 pH-down(산) / pH-up(염기) |
| **성분 드리프트** | 센서로 쫓지 말고 **운용 규칙(교체 주기 or 배액식)**으로 관리 |

### 4.5 드리프트 관리 — 실제 상업 재배는 어떻게 하나

성분 드리프트는 측정해서 교정하는 것이 아니라 **운용 방식**으로 다룬다. 상업 재배는 크게 두 갈래다:

| 방식 | 내용 | 드리프트 대응 |
| --- | --- | --- |
| **순환식 (recirculating)** | 양액을 돌려 씀 (우리 NFT가 여기 해당) | **1~3개월마다 전량 배출·교체**로 리셋. 그 사이에는 배출수(runoff) EC·pH를 측정하며 물·양액 보충 |
| **배액식 (drain-to-waste)** | 매 관수마다 신선한 양액 공급, 남는 10~20%는 폐기 | 순환을 안 하므로 **드리프트·pH 편차가 애초에 안 생김**. 대규모 상업 온실에서 흔함. 대신 물·양액 소비가 큼 |

> ⚠️ 흔한 오해: "상업 재배는 다 전량 교체한다"는 과장이다. 대규모는 오히려 **배액식으로 드리프트를 회피**하는 경우가 많고, 순환식도 **주기가 몇 주가 아니라 1~3개월**이며 평소엔 보충·모니터링으로 버틴다.

**우리 시스템(소규모 상추 NFT, 순환식) 권고**: 1~3개월 주기 전량 교체 + 그 사이 EC 기준 정배합 보충. 배액식은 물·양액 낭비가 커 소규모엔 부적합.

> 참고 신호: 상추가 질산태 질소를 흡수하면 탱크 pH가 서서히 **상승**한다. pH 상승은 "흡수가 활발함"의 대략적 지표는 되지만, 전체 성분 비율을 복원할 정도는 아니다.

## 5. 이온 선택성(NPK) 센서를 쓸 수 없는 이유

### 5.1 싼 "NPK 센서"(₩3만~10만)는 사실상 가짜

- 시중 저가 토양 NPK 센서(JXCT류, RS485/Arduino)는 **N·P·K를 직접 재지 않고 EC로 추정**한다. 이미 가진 EC 센서와 본질이 같다.
- 질소 이온과 칼슘·나트륨·철을 **구분하지 못한다** → "칼슘 누적 vs 인산 고갈"에 무용지물.
- 게다가 **토양용**이지 양액용이 아니다.

### 5.2 진짜 ISE는 이온당 100만 원대

| 센서 | 대략 가격 |
| --- | --- |
| 질산(NO₃⁻) ISE (YSI TruLine) | ~$925 (≈125만원) |
| 칼륨(K⁺) ISE (YSI TruLine) | ~$1,142 (≈155만원) |
| 질산 ISE (Hach Intellical) | ~$1,402 (≈190만원) |
| 전극 교체 모듈(소모품) | ~$330 |

+ 전용 트랜스미터/미터 별도. 최소 구성(질산+칼륨)만 해도 **300만~400만 원**.

### 5.3 결정적 함정 — 인산(P)은 실시간 측정 수단이 없다

우리 드리프트의 핵심 성분이 **인산(고갈)·칼슘(누적)**인데:
- **인산은 신뢰할 만한 ISE가 없다.** P는 시약 발색법(랩 분석)으로 재는 물질이라 실시간 프로브 자체가 없다.
- 질산·칼륨에 각각 150만 원을 써도 **정작 궁금한 인산은 여전히 못 본다.**

### 5.4 결론

- 완전한 성분 제어는 이 예산으로도 **인산 때문에 구조적으로 불가능**.
- 상추 NFT 규모에서 이온 센서는 **비용 대비 실익이 거의 없다.** 대규모 연구·상업 온실급에서나 투자 회수가 된다.

## 6. 종합 결론 및 권고

1. **도징 계수는 확보 완료** — 실험 1~4로 6개 계수를 도출했고 재현성도 합격. `config.py`에 반영하면 즉시 실전 적용 가능.
2. **pH는 양액으로 올릴 수 없다** — 시스템은 pH를 A·B로 능동 제어하지 말고, 하강만 활용하거나 별도 산/염기로 처리.
3. **도징의 목표를 EC 유지로 좁힐 것** — A:B = 1:1 정배합으로 EC를 맞추는 것이 밸런스를 지키는 유일한 방법. pH를 쫓아 비율을 흔들면 오히려 해롭다.
4. **성분 밸런스는 센서가 아니라 운용 규칙으로** — 순환식 기준 1~3개월 주기 전량 교체 + 평소 EC 보충·상추 상태 관찰(대규모라면 배액식도 대안). 이온 센서 투자는 권하지 않음.
5. **현 EC·pH 하드웨어로 할 수 있는 것과 없는 것의 경계를 명확히** — 할 수 있는 것: EC 유지, pH 하강 보조, 이상 감지. 할 수 없는 것: 성분별 밸런스 실시간 제어.

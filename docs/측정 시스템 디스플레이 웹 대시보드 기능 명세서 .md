# 측정 시스템 디스플레이 웹 대시보드 기능 명세서

> 작성일: 2025-06
> 
> 
> 범위: Raspberry Pi 4B 기반 백엔드 서버 + 웹 대시보드 프론트엔드
> 
> 전제: 아두이노에서 시리얼 통신(UART)으로 JSON 포맷 데이터가 주기적으로 수신됨
> 

---

## 1. 시스템 개요

### 1.1 전체 아키텍처

```
[Arduino]
    │
    │  UART (Serial, JSON 포맷)
    ▼
[Raspberry Pi 4B]
    ├── FastAPI 백엔드 (uvicorn)
    │     ├── Serial Reader (asyncio background task)
    │     ├── SQLite (aiosqlite)
    │     └── WebSocket Broadcaster
    │
    └── Chromium (키오스크 모드)
          └── 웹 대시보드 (Vanilla JS + Chart.js)
              └── 같은 WiFi 내 외부 브라우저에서도 접근 가능
```

### 1.2 데이터 흐름

```
Arduino (30초 주기)
    └─ {"ec": 1.85, "ph": 6.2, "temp": 23.4, "ts": 1719123456}
         │
         ▼
    FastAPI Serial Reader
         ├─ 파싱 및 유효성 검증
         ├─ SQLite 저장
         └─ WebSocket broadcast
                  │
                  ▼
         브라우저 (실시간 업데이트)
```

## 2. 기술 스택

| 영역 | 기술 | 선택 이유 |
| --- | --- | --- |
| 웹 프레임워크 | **FastAPI** | 비동기 지원, WebSocket 내장, 경량 |
| ASGI 서버 | **uvicorn** | RPi 4B에서 낮은 메모리 사용 |
| 시리얼 통신 | **pyserial-asyncio** | blocking I/O 없이 async 시리얼 수신 |
| 데이터베이스 | **SQLite + aiosqlite** | 별도 DB 서버 불필요, 비동기 지원 |
| 프론트엔드 | **Vanilla JS** | 빌드 과정 없음, RPi에서 npm build 불필요 |
| 차트 | **Chart.js (CDN)** | script 태그 하나로 사용, 경량 |
| 브라우저 | **Chromium (키오스크 모드)** | RPi OS 기본 탑재 |

### 2.1 Python 의존성

```bash
pip install fastapi uvicorn aiosqlite pyserial-asyncio
# 이게 전부
```

## 3. 기능 명세

### 3.1 시리얼 수신 및 파싱

**기능 ID:** F-01

**담당 모듈:** `serial_reader.py`

| 항목 | 내용 |
| --- | --- |
| 입력 포맷 | JSON 문자열 (개행 구분) |
| 예시 입력 | `{"ec": 1.85, "ph": 6.2, "temp": 23.4, "ts": 1719123456}` |
| 수신 주기 | 아두이노 전송 주기에 종속 (약 30초) |
| 파싱 실패 시 | 해당 줄 무시 후 로그 기록, 프로세스 중단 없음 |
| 유효성 검증 | ec, ph, temp 값이 숫자형인지 확인 |

**유효 범위 (이상값 필터링 기준)**

| 항목 | 정상 범위 | 비고 |
| --- | --- | --- |
| EC | 0.5 ~ 3.5 mS/cm | 상추 기준 |
| pH | 5.5 ~ 7.5 | 수경재배 허용 범위 |
| Temp | 5.0 ~ 40.0 °C | 센서 오작동 필터 |

---

### 3.2 데이터 저장 (SQLite)

**기능 ID:** F-02

**담당 모듈:** `database.py`

**테이블 스키마: `sensor_logs`**

```sql
CREATE TABLE sensor_logs (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    ec        REAL    NOT NULL,
    ph        REAL    NOT NULL,
    temp      REAL    NOT NULL,
    timestamp INTEGER NOT NULL,   -- Unix timestamp (아두이노 전송값 or 수신 시각)
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

**저장 정책**

| 항목 | 내용 |
| --- | --- |
| 저장 시점 | 시리얼 수신 즉시 (유효성 검증 통과 후) |
| 보관 기간 | 최근 30일 (초과분 자동 삭제 — 별도 cleanup task) |
| 히스토리 조회 | 최근 N개 레코드 API 제공 |

---

### 3.3 WebSocket 브로드캐스트

**기능 ID:** F-03

**담당 모듈:** `websocket_manager.py`

| 항목 | 내용 |
| --- | --- |
| 엔드포인트 | `ws://[RPi IP]:8000/ws` |
| 브로드캐스트 시점 | 새 센서값 수신 즉시 |
| 연결 클라이언트 수 | 복수 연결 지원 (미니 디스플레이 + 외부 폰/PC 동시) |
| 연결 끊김 처리 | 클라이언트 목록에서 자동 제거, 서버 프로세스 영향 없음 |

**브로드캐스트 메시지 포맷**

```json
{
  "ec": 1.85,
  "ph": 6.2,
  "temp": 23.4,
  "timestamp": 1719123456,
  "status": {
    "ec": "normal",
    "ph": "warning",
    "temp": "normal"
  }
}
```

`status` 값: `"normal"` / `"warning"` / `"danger"`

### 3.4 REST API

**기능 ID:** F-04

| 메서드 | 엔드포인트 | 설명 |
| --- | --- | --- |
| GET | `/api/latest` | 최신 센서값 1건 반환 |
| GET | `/api/history?limit=100` | 최근 N개 히스토리 반환 |
| GET | `/api/status` | 서버 상태 및 시리얼 연결 상태 확인 |
| GET | `/` | 대시보드 HTML 서빙 |

---

### 3.5 웹 대시보드 (프론트엔드)

**기능 ID:** F-05

**구현:** `static/index.html` (단일 파일)

**화면 구성**

```
┌─────────────────────────────────────┐
│  NFT 배양액 모니터             [상태] │
├───────────┬────────────┬────────────┤
│    EC      │     pH     │    Temp    │
│  1.85      │    6.2     │   23.4°C   │
│ mS/cm      │            │            │
│ [정상]     │  [주의]    │  [정상]    │
├─────────────────────────────────────┤
│  EC 추이 그래프 (Chart.js)           │
├─────────────────────────────────────┤
│  pH 추이 그래프 (Chart.js)           │
└─────────────────────────────────────┘
```

**상태 표시 색상**

| 상태 | 색상 | 조건 |
| --- | --- | --- |
| 정상 (normal) | 초록 | 정상 범위 내 |
| 주의 (warning) | 노랑 | 허용 범위 경계 ±10% |
| 위험 (danger) | 빨강 | 허용 범위 초과 |

**WebSocket 동작**

- 페이지 로드 시 자동 연결
- 실시간 수신 시 카드 값 즉시 업데이트
- 그래프에 최근 50개 포인트 표시
- 연결 끊김 시 5초 후 자동 재연결

## 4. 개발 단계 (Phase)

| Phase | 목표 | 완료 기준 |
| --- | --- | --- |
| **1-A** | FastAPI 서버 기동 + 더미 데이터 생성기 | `/api/latest` 호출 시 더미값 반환 |
| **1-B** | WebSocket 브로드캐스트 + 프론트 연결 | 브라우저에서 실시간 값 변화 확인 |
| **1-C** | SQLite 저장 + `/api/history` | DB에 데이터 쌓이고 조회 가능 |
| **2-A** | 시리얼 수신 연동 (더미 → 실제 아두이노) | 아두이노 값이 대시보드에 표시 |
| **2-B** | 상태 판정 로직 + 색상 표시 | 범위 이탈 시 경고 색상 표시 |
| **3** | 키오스크 모드 자동 실행 설정 | RPi 부팅 시 대시보드 자동 표시 |

## 5. 파일 구조

```
nft-monitor/
├── main.py                  # FastAPI 앱 진입점, lifespan 설정
├── serial_reader.py         # 시리얼 수신 asyncio 태스크

└── data/

```

## 6. 설정값 (config.py)

```python
# 시리얼 설정
SERIAL_PORT = "/dev/ttyUSB0"   # 아두이노 연결 포트
SERIAL_BAUD = 9600

# 서버 설정
HOST = "0.0.0.0"               # 외부 접근 허용
PORT = 8000

# 센서 정상 범위
EC_RANGE   = (0.5, 3.5)        # mS/cm
PH_RANGE   = (5.5, 7.5)
TEMP_RANGE = (5.0, 40.0)       # °C

# 데이터 보관
MAX_HISTORY_DAYS = 30
CHART_POINTS = 50              # 그래프 표시 포인트 수
```

## 7. 키오스크 자동 실행 (Phase 3)

RPi 부팅 시 서버 + 브라우저 자동 실행

```bash
# /etc/systemd/system/nft-monitor.service
[Unit]
Description=NFT Monitor Server
After=network.target

[Service]
ExecStart=uvicorn main:app --host 0.0.0.0 --port 8000
WorkingDirectory=/home/pi/nft-monitor
Restart=always

[Install]
WantedBy=multi-user.target
```

```bash
# ~/.config/autostart/kiosk.desktop
[Desktop Entry]
Type=Application
Exec=chromium-browser --kiosk --noerrdialogs http://localhost:8000
```

## 8. 미결 사항 (센서 확정 후 결정)

| 항목 | 내용 |
| --- | --- |
| 시리얼 포트 | 아두이노 연결 시 `/dev/ttyUSB0` or `/dev/ttyACM0` 확인 필요 |
| 전송 주기 | 아두이노 측 샘플링 주기 확정 필요 (현재 30초 가정) |
| 타임스탬프 | 아두이노가 ts를 보내지 않을 경우 RPi 수신 시각 사용 |
| 온도 보상 | EC/pH 온도 보상 계산을 아두이노에서 할지 RPi에서 할지 결정 필요 |
